# Submission Kit

Use this pack when submitting `markoblogo/abvx-agent-skills` to curated lists, agent-skill catalogs, or install surfaces.

## Primary Message

Do not position the repo as "50+ skills for everything."

Position it as:

> A small, auditable skillpack for coding agents that helps them produce smaller diffs, debug from evidence, reduce token waste, and verify work before saying done.

## One-Line Description

Small, auditable agent skills for coding work: smaller diffs, evidence-led debugging, token control, and real verification.

## Two-Line Description

ABVX Agent Skills is a reviewable skillpack for Codex-style agents. It focuses on narrower implementation, debugging from evidence, token economy, and browser/verification gates instead of generic prompt bloat.

## Three-Line Description

ABVX Agent Skills is a small, auditable skillpack for coding agents. The pack is optimized around practical project work: write smaller patches, debug from evidence, compress shell noise, and verify work before claiming success. Each skill is packaged as compact `SKILL.md` instructions with validation, metadata, and a repo-native catalog.

## Top 5 Skills To Start With

1. `minimal-diff-builder` - smallest correct implementation path
2. `diagnose` - disciplined evidence-led debugging
3. `token-efficient-execution` - lower waste in long coding sessions
4. `rtk-assisted-shell` - compact shell-heavy workflows
5. `browser-verification` - verify real browser behavior before "done"

## Starter Packs

- `debugging`: `diagnose`, `repo-debugging-ledger`, `graph-guided-code-reading`
- `small-diffs`: `minimal-diff-builder`, `delivery-preflight-gate`
- `token-economy`: `rtk-assisted-shell`, `shell-output-compaction`, `token-efficient-execution`
- `frontend-verification`: `browser-verification`, `design-critique-polish`

## Trust Markers

- validation workflow on `main`
- static security audit workflow on `main`
- `gh skill publish --dry-run` readiness
- generated `CATALOG.md` plus live searchable catalog on ABVX Lab
