# Skill Card: designmd-brand-kit

## Description
Creates, inspects, and applies DESIGN.md brand kits for brand-aware frontend work and client research.

## Owner
ABVX / Anton Biletskiy-Volokh

## License
MIT. See repository LICENSE.

## Intended Use
Use for landing pages, frontend UI, pitch pages, redesigns, client audits, and potential-client research where website identity should shape the output.

## Out of Scope
Do not use to clone a third-party site one-to-one. Do not publish client-facing materials or send outreach without approval.

## Sources and Attribution
Inspired by context-dot-dev `designmd-supply` and the DESIGN.md design-system artifact pattern.

## Inputs and Outputs
Inputs: URL, domain, DESIGN.md, screenshots, CSS/Tailwind tokens, or client notes.

Outputs: brand constraints, DESIGN.md files, Tailwind v4 theme tokens, CSS variables, UI implementation notes.

## Risks and Mitigations
- Risk: extracted tokens are wrong or incomplete. Mitigation: verify against screenshots/site evidence.
- Risk: brand imitation. Mitigation: use compatible constraints, not copying.
- Risk: privacy in client research. Mitigation: do not store private contacts or sensitive notes in reusable skills.

## Evaluation
Evaluated by structural validation and manual review against frontend, pitch-page, redesign, and client-audit use cases.
