# Frontend Taste Layer Pilot

Date: 2026-07-19

The `frontend-taste-layer` 0.2 contract was checked against two independent production surfaces. Browser captures were temporary and are not distributed with the skill.

## ABVX Lab skill catalog

- Design read: searchable developer-tool catalog for technical evaluators, using a restrained documentation language.
- Relative axes: low composition variance, no material motion, high information density.
- Preserve: search/filter behavior, category grouping, source links, semantic HTML, and the static-site architecture.
- Browser evidence: 1440×1000 and 390×844; no horizontal overflow at 390px; reduced-motion emulation active.
- Accepted correction: add an explicit existing ABVX favicon to stop the catalog page requesting missing `/favicon.ico`.
- Rejected: command palette, alternate view modes, and pinboard rail. They add product behavior beyond a bounded taste review.
- Lazyweb evidence report: <https://www.lazyweb.com/report/lazyweb/2a41ac0c-d49e-4e4f-b74c-d4b52377c48a/?source=create>

## ABVX Press `/books`

- Design read: editorial publishing catalog for readers browsing official lines, standalone books, and companion systems, using a high-contrast print-catalog language.
- Relative axes: high section-level composition variance, restrained motion, high catalog density.
- Preserve: source-faithful covers, titles and language editions, publishing-line hierarchy, links, and ABVX Press typography.
- Browser evidence: 1440×1000 and 390×844; no horizontal overflow at 390px; reduced-motion emulation active.
- Accepted correction: none. Repeated record grids are justified by comparison and differentiated by publishing-line headers, source artwork, and section scale.
- Rejected: edition folding, pagination, scroll minimap, and an additional sticky navigation layer. They change content semantics or interaction scope without owner evidence that navigation is failing.
- Lazyweb evidence report: <https://www.lazyweb.com/report/lazyweb/c4029436-8346-48ea-9ab3-ff0e24d56d26/?source=create>

## Result

The contract activated correctly for marketing/editorial surfaces, preserved project-specific assets and hierarchy, and rejected product-feature expansion. Product UI remains routed to Lazyweb and UX review rather than inheriting these taste heuristics as universal policy.
